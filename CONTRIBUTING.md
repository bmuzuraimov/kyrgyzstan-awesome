# Руководство по участию

Спасибо за желание улучшить этот список!

---

## Требования

Прежде чем предложить проект, убедитесь, что он соответствует всем критериям:

- **Открытый исходный код** — репозиторий должен быть публично доступен.
- **Связан с Кыргызстаном** — создан кыргызстанцами, о Кыргызстане, или используется в Кыргызстане.
- **Значимый** — проект оказал реальное влияние: используется людьми, решает настоящую проблему, или является важным ресурсом для экосистемы — вне зависимости от даты последнего коммита. Заброшенный проект с нулевым влиянием включён не будет; влиятельный проект, MVP которого завершена, — будет.
- **Задокументирован** — наличие README с описанием цели проекта обязательно.
- **Лицензирован** — проект должен иметь явно указанную лицензию с открытым исходным кодом.

## Как добавить проект

1. **Создайте форк** репозитория.
2. **Создайте ветку** с именем вроде `add-название-проекта`.
3. **Добавьте ваш проект** в соответствующую категорию README.md, соблюдая формат.
4. **Отправьте pull request** с заголовком `Add Название Проекта`.
5. В комментарии к PR напишите слово `unicorn` — это подтверждает, что вы прочитали это руководство.

## Формат записи

Каждая запись должна строго соответствовать следующему формату:

```markdown
- [Название проекта](https://github.com/org/repo#readme) — Краткое описание на русском.
```

Правила форматирования:

- Название проекта пишется с заглавной буквы каждого значимого слова.
- Описание начинается с заглавной буквы и заканчивается точкой.
- Ссылка ведёт на `#readme` файл репозитория для немедленного отображения документации.
- Описание должно быть объективным — не рекламным слоганом.
- Длина описания: не более 120 символов.

## Категории

Используйте существующие категории. Если ваш проект не попадает ни в одну из них, предложите новую в PR с обоснованием.

| Категория | Описание |
|---|---|
| Инфраструктура и DevOps | DevOps, библиотеки, CI/CD |
| Геолокация и картография | Карты, геокодинг, навигация |
| Обработка кыргызского языка (NLP) | Модели, датасеты, инструменты |
| Открытые CMS и платформы | Системы управления контентом |
| Гражданские технологии | Прозрачность, гражданское участие |
| Открытые данные и API государства | Порталы данных, обёртки API |
| Образование и EdTech | Обучение, курсы, школьные инструменты |
| Финансовые технологии и платежи | Платежи, переводы, открытый банкинг |
| Здравоохранение | Медицинские системы, телемедицина |
| Сельское хозяйство и природные ресурсы | Агротех, мониторинг земель |
| Кибербезопасность | OSINT, безопасность, CTF |
| ИИ-инструменты и автоматизация | ИИ-агенты, LLM, автоматизация |
